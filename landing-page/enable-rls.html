Fix Supabase Row Level Security (RLS) Issue

The error message "new row violates row-level security policy for table 'waitlist'" indicates that Row Level Security is enabled on your waitlist table, but there's no policy that allows anonymous users to insert data.

Option 1: Disable RLS for the waitlist table (Simplest)

If you don't need row-level security for your waitlist table, you can simply disable it:

-- Run this in your Supabase SQL Editor
ALTER TABLE waitlist DISABLE ROW LEVEL SECURITY;

Option 2: Create a policy to allow anonymous inserts (More secure)

If you want to keep RLS enabled but allow anonymous users to insert their emails:

-- Run this in your Supabase SQL Editor
-- First, ensure RLS is enabled
ALTER TABLE waitlist ENABLE ROW LEVEL SECURITY;

-- Create a policy that allows anyone to insert data
CREATE POLICY "Allow anonymous inserts" 
ON waitlist 
FOR INSERT 
TO anon
WITH CHECK (true);

-- Optionally, if you want to allow users to see their own submissions
CREATE POLICY "Allow users to see their own submissions" 
ON waitlist 
FOR SELECT 
TO anon
USING (true);

How to Apply These Changes

  1. Go to your Supabase Dashboard
  2. Select your project
  3. Go to the SQL Editor (left sidebar)
  4. Create a new query
  5. Paste one of the SQL snippets above
  6. Run the query

After applying these changes, your waitlist form should work correctly and be able to insert data into the waitlist table.
